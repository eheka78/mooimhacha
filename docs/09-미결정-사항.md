# 09. 미결정 사항

구현 착수 전 또는 PoC 후 결정이 필요한 항목.

## 정책 결정 (우선)

- [ ] 아젠다 생성 AI 레벨 (룰 / 하이브리드 / LLM 중심) — **전체 구현 일정에 영향**
- [ ] 진척도 평가 방식 디테일 (안건 완료율 자동 계산 + 사용자 보정 범위)
- [x] **백엔드 프레임워크 → NestJS 결정 (2026-05-20)**
- [x] **인프라 → AWS (Start AWS 프로그램) 결정 (2026-05-20)** — 상세는 [10-AWS-인프라-제약.md](10-AWS-인프라-제약.md)
- [x] **캐시 전략 → ElastiCache 권한 없음. NestJS 인메모리 Map + 5초 RDS flush로 결정 (2026-05-20)**
- [ ] **DB 접근 방식 (ORM vs 쿼리 빌더 vs raw SQL) 및 마이그레이션 도구** — 팀원 구성·역할 협의 후 결정. 후보: TypeORM / Prisma / Kysely / Drizzle / `mysql2` raw SQL
- [ ] 사유 결석 인정 권한 (자동 / 팀장 승인) — **데이터 모델에 영향**

## STT 관련 (Moonshine)

- [ ] Moonshine 모델 크기 선택 (`tiny` vs `base`) — PoC 정확도·성능 결과로 결정
- [ ] VAD 임계값 선택 — 라이브러리는 PoC에서 `@ricky0123/vad-web`(Silero VAD) 채택, 임계값은 PoC 후 결정
- [ ] `confidence` 산출 방식 — Moonshine ONNX에서 신뢰도 점수를 어떻게 얻을지
- [ ] 모델 패키징 방식 (앱 동봉 vs 최초 실행 시 다운로드)

## 1주차 PoC 후 결정

- [ ] 교수 제출용 PDF P0 여부
- [ ] 신뢰도 등급 정확한 임계값
- [ ] 마감 준수 점수 곡선 수치

## 디테일 미설계

- [ ] 회의 외 시간 대시보드 레이아웃
- [ ] 회의 후 정리 화면 디테일
- [ ] 회의 전 아젠다 검토 화면 디테일
- [ ] DB 스키마 상세 정의
- [ ] 미접속자 경고 정책 디테일

## 수치 확정

- [ ] 글자수 상한 (현재 500자)
- [ ] 누적 점수 가중 방식 옵션 확정 (시간 가중 추천)
- [ ] 태스크 완료 축 내부 배분 (액션 완료율 vs 마감 준수율 비율)
- [ ] 3축 기본 가중치(태스크 50 / 발언 30 / 참석 10)의 팀별 조정 허용 범위

## 기타

- [ ] 회의 무효 처리 기준
- [ ] 테스트 모드 회의 (누적 미반영 옵션)
- [ ] 마감일 자동 기본값 (다음 회의 시작 vs 회의 종료 후 N일)
- [ ] 데이터 삭제 요청 권한 (본인 / 팀장)
- [ ] 누적 점수 결석 제외 정책 디테일
- [ ] 외부 게스트 입장 P0 여부
- [ ] 최소 회의 길이 (5분 미만 누적 미반영)

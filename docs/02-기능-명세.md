# 02. 기능 명세

구현 대상 기능. (★) = 핵심 차별화 기능. 우선순위는 [08](08-우선순위-로드맵.md) 참조.

## 회의 전

### 인증
- 카카오 로그인(OAuth) — 별도 회원가입 없이 간편 인증
- 최초 로그인 시 대학교·학과 입력 (선택, 향후 통계용)
- 백엔드에서 카카오 토큰 검증 후 자체 세션/JWT 발급

### 팀·프로젝트
- 팀 생성, 초대 코드로 합류 — 코드는 생성·만료 설정 가능, 초대 권한은 팀장
- 역할 구분: 팀장 / 팀원
- 권장 팀 규모: 4명 이하 (대학생 팀플 기준)
- 3계층 목표 구조: **프로젝트 > 마일스톤 > 회의별 목표**
- 마일스톤 진척도 = 하위 안건 완료율로 자동 계산

### 회의 생성
- 입력: 일시, 총 예상 시간, 주제(선택)
- 백엔드 CRUD

### 아젠다
- 회의 안건은 **직전 회의 종료 후 LLM이 생성** (아래 회의 후 §다음 회의 준비 참조)
- **첫 회의는 사전 아젠다 없이 시작** — 회의 중 즉석 안건 추가로 진행
- 2회차부터는 직전 회의 결과로 생성된 아젠다를 팀장이 검토·수정 후 팀원에게 배포

### 사전 의견 수집 (P1)
- 안건별 의견·자료 사전 제출 폼

### 회의 시작 알림
- 인앱: Electron Notification API
- 앱 미실행 시: Web Push API
- 백업: 이메일
- 트리거: 회의 시작 5분 전

## 회의 중 — 보조 창

### 보조 창 구성
- 폭 400px 고정, 높이 가변
- Always-on-top 옵션
- 창 위치 기억 (electron-store)
- 영역 비율: 헤더(시간·참여자) 8% / **안건 추적 40%** / **기여도 바 30%** / 결정·액션 입력 20% / 최근 항목 10%

### 회의 룸
- WebSocket 기반 다인 접속
- 미접속자 있어도 경고 후 시작 가능 (유연 정책)
- 시각 동기화: 서버가 T0를 broadcast, 클라이언트는 상대 시간 사용

### 마이크 권한·동의
- `getUserMedia()`로 마이크 접근
- 회의 시작 전 전원 각자 녹음 동의

### STT 처리 (로컬)
- 캡처된 오디오를 VAD로 발화 구간 분할 → Moonshine 로컬 추론 → 텍스트
- 발화 단위로 텍스트만 서버 전송 (상세: [05](05-STT-음성-처리.md))

### 안건 추적 (★)
- 회의 전 아젠다 자동 연결 + 회의 중 즉석 안건 추가
- 상태 3단계: 대기 / 진행 중 / 완료
- 누구나 상태 변경 가능, 변경 시 broadcast 알림
- 시간 초과 시각화: 100% 색 변경 → 150% 알림 → 200% 강한 신호
- 발화 자동 매칭: 각 utterance에 현재 진행 중 `agenda_id` 부여

### 안건별 요약
- 안건이 **'완료'로 전환될 때** GPT-4o-mini가 해당 안건의 발화를 요약 (안건당 1회)
- 요약은 LLM 응답 후 비동기로 도착, WebSocket broadcast
- **요청 시에만 표시** — 완료된 안건 우측에 '요약 보기' 버튼이 생기고, 버튼을 누르면 해당 안건 아래로 요약이 펼쳐짐 (기본 접힘, 다시 누르면 접힘)
- 진행 중·대기 안건에는 버튼 미표시
- 회의록 전체는 회의 중 표시하지 않음(아래 회의록 참조) — 회의 중 노출되는 것은 사용자가 펼친 안건 요약뿐
- 생성된 요약은 회의 후 AI 회의 종합 정리(아래)의 입력으로 재사용

### 실시간 기여도 바 (★)
- 두 축을 별개로 표시: 역할 수행도 / 발언 비중
- 발언 비중 = 글자수 비율 (참여 시간 보정 — [06](06-기여도-산정.md) 공식)
- WebSocket broadcast, 0.5초 디바운스
- Framer Motion 막대 애니메이션
- 막대 = 비율(%), 라벨 = 절대 글자수
- 발화 중 사용자에 🎤 표시
- 침묵 알림은 본인에게만 표시
- ⓘ 아이콘으로 "추정치" 안내

### 결정사항·액션 빠른 입력
- 결정사항: 한 줄 텍스트 + Enter
- 액션 아이템 필수 필드: 담당자 / 내용 / 마감일
- 액션 선택 플래그: "다음 회의 전까지"
- 누구나 입력 가능
- 단축키: `Ctrl+D` 결정, `Ctrl+A` 액션

### 회의록
- 회의 중에는 표시하지 않음
- utterance만 누적, 회의 종료 시 그루핑 알고리즘 일괄 적용

## 회의 후 — 풀 화면

### 정리 화면
- 보조 창을 닫고 풀 화면 새 BrowserWindow 오픈
- 콘텐츠: 회의록(안건별 그루핑) + 결정사항 + 액션 + 기여도 대시보드

### 회의록 그루핑 알고리즘
- 시간순 정렬
- 3초 이내 연속 발화는 하나로 묶음
- 짧은 발화는 들여쓰기
- `agenda_id` 기준 안건별 자동 분류

### AI 회의 종합 정리
- GPT-4o-mini 회의당 1회 호출 (회의 후 1번째 호출)
- 입력: 안건별 요약 + 회의록 + 수동 입력된 결정·액션
- 출력: 회의 요약, 누락된 결정사항, 정리된 태스크(액션 아이템) — 각 항목에 출처 utterance ID 포함

### 검토·확정
- 권한: 작성자 본인 또는 팀장
- 수정·삭제·확정 가능
- 확정 후 액션 담당자에게 알림
- Fallback: 24시간 경과 시 자동 확정

### 대시보드
- 기여도: 태스크 완료 / 발언 비중 / 참석 3축 + 종합 점수 표시 ([06](06-기여도-산정.md))
- 액션 아이템: 담당자별·마감일별 정렬

### 다음 회의 준비
- `PreparationItem` 입력
- 마일스톤 진척도 보정
- **다음 회의 아젠다 생성** — GPT-4o-mini 회의당 1회 호출 (회의 후 2번째 호출)
  - 입력: 이번 회의 종합 결과(결정·태스크) + 미해결 액션 + 마일스톤 진척 + `PreparationItem`
  - 출력: JSON (안건 목록, 안건별 예상 시간, 출처 라벨)
  - 팀장이 검토·수정 후 팀원에게 배포
  - AI 개입 수준(룰/하이브리드/LLM 중심)은 미확정 ([09](09-미결정-사항.md))

### 교수 제출용 PDF (P0/P1 미정)
- 서버 사이드(Puppeteer/ReportLab) 또는 클라이언트(jsPDF)
- 내용: 날짜·참석자·안건별 회의록·결정사항·역할 분담

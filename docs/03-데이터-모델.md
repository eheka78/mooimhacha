# 03. 데이터 모델

엔티티별 필드 정의. DB 스키마 상세(타입·인덱스·제약)는 미확정 — [09](09-미결정-사항.md) 참조.

## 사용자·팀

### User
`id`, `kakao_id`, `email`, `name`, `university`, `department`

### Team
`id`, `name`, `course_name`, `created_by`

### TeamMembership
`team_id`, `user_id`, `role`(leader/member), `joined_at`

### TeamSettings
`team_id`, `contribution_weights`(태스크/발언/참석), `min_participation_ratio`, `punctuality_grace_ratio`, `max_utterance_chars`, `presence_grace_seconds`, `absent_meeting_handling`, `deadline_penalty_curve`, `contribution_visibility`

## 프로젝트·목표

### Project
`id`, `team_id`, `title`, `status`

### Milestone
`id`, `project_id`, `title`, `deadline`, `order`, `progress_ratio`

## 회의

### Meeting
`id`, `team_id`, `scheduled_at`, `total_minutes`, `status`, `t0_timestamp`,
`summary`(회의 후 AI 종합 정리로 생성)

### Agenda (★)
`id`, `meeting_id`, `title`, `estimated_minutes`, `order`,
`status`(pending/active/done), `started_at_offset_ms`, `ended_at_offset_ms`,
`actual_minutes`, `source`(ai_recommended/manual/ad_hoc), `summary`(완료 시 LLM 생성), `milestone_id`

### Utterance
`id`, `meeting_id`, `user_id`, `text`, `char_count`, `confidence`,
`started_at_offset_ms`, `ended_at_offset_ms`, `agenda_id`(★ 안건 매칭)

## 회의 산출물

### Decision
`id`, `meeting_id`, `content`, `created_by`, `source`(manual/ai_extracted),
`source_utterance_id`, `confirmed`

### ActionItem
`id`, `meeting_id`, `assignee_id`, `description`, `due_date`,
`is_for_next_meeting`, `status`, `source`, `source_utterance_id`, `confirmed`

## 측정·이벤트

### PresenceEvent
`id`, `user_id`, `meeting_id`, `event_type`(join/leave/disconnect/reconnect),
`timestamp_offset_ms`, `reason`(user_action/network/browser_close/timeout)

### AnomalyEvent
`id`, `user_id`, `meeting_id`, `event_type`, `timestamp_offset_ms`, `severity`, `metadata`

### ContributionScore
`id`, `user_id`, `meeting_id`,
`task_score`(태스크 완료 축), `speech_score`(발언 비중 축), `attendance_score`(참석 축),
`action_completion_ratio`, `deadline_adherence`, `speech_ratio`,
`attendance_ratio`, `punctuality_score`,
`composite_score`, `confidence_level`, `excluded_axes`

> `task_score`/`speech_score`/`attendance_score`는 3축 점수, 나머지 비율 필드는 각 축의 산정 근거([06](06-기여도-산정.md)).

## 시간 표현 규칙

- 회의 내 모든 시각은 서버 T0 기준 **상대 오프셋(`_offset_ms`)** 으로 저장
- 절대 시각은 `scheduled_at`, `t0_timestamp`, `due_date` 등에만 사용

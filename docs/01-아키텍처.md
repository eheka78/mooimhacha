# 01. 아키텍처 & 기술 스택

## 데이터 흐름

```
[각 사용자의 Electron]
  마이크 → getUserMedia 오디오 캡처
        → VAD로 발화 구간 분할
        → Moonshine ONNX 로컬 추론 (음성 → 텍스트)
        → 발화 단위로 텍스트만 서버 전송  (음성은 기기 밖으로 나가지 않음)
              ↓
        [서버 — EC2 단일 인스턴스의 NestJS]
          ├─ MySQL(RDS)에 utterance 저장
          ├─ 인메모리 Map에 사용자별 누적 글자수 갱신
          │    └─ 5초마다 RDS에 flush
          └─ WebSocket broadcast (0.5초 디바운스)
              ↓
[모든 사용자의 Electron]
  Framer Motion으로 기여도 바 갱신
```

핵심: STT 전 과정을 **클라이언트 로컬에서** 수행. 음성은 기기를 떠나지 않고, 서버로는 **텍스트만** 전송. 서버는 저장·집계·브로드캐스트만 담당.

## 클라이언트 (Electron 데스크탑)

| 영역           | 기술                                                    |
| -------------- | ------------------------------------------------------- |
| 프레임워크     | Electron                                                |
| 렌더러         | React + TypeScript                                      |
| 메인 프로세스  | Node.js                                                 |
| UI             | TailwindCSS + shadcn/ui                                 |
| 상태관리       | Zustand 또는 React Query                                |
| 애니메이션     | Framer Motion                                           |
| STT 추론       | Moonshine ONNX (`onnxruntime-web` 또는 transformers.js) |
| 음성 구간 분할 | VAD (예: Silero VAD)                                    |
| 빌드           | electron-builder                                        |
| 로컬 저장      | electron-store                                          |

> **PoC 메모 (2026-05-18)**: STT PoC 단계에서 추론은 transformers.js, VAD는 `@ricky0123/vad-web`(Silero VAD)를 채택. 최종 확정은 1주차 PoC 검증 후 — [progress/2026-05-18-STT-PoC-파이프라인.md](../progress/2026-05-18-STT-PoC-파이프라인.md)

## 백엔드

| 영역        | 기술                                                            |
| ----------- | --------------------------------------------------------------- |
| 서버        | NestJS (Node.js)                                                |
| DB          | MySQL (RDS 프리티어)                                            |
| DB 접근     | 미결정 — 팀원 협의 후 결정 ([09 참조](09-미결정-사항.md))         |
| 캐시        | NestJS 프로세스 인메모리 Map (5초마다 RDS flush) — Redis 미사용 |
| 실시간 통신 | WebSocket (NestJS Gateway 직접 서빙, 단일 EC2)                  |

## 외부 API

| 영역 | 서비스                          | 용도                      | 비고                                       |
| ---- | ------------------------------- | ------------------------- | ------------------------------------------ |
| STT  | Moonshine (`moonshine-tiny-ko`) | 음성 → 텍스트             | 클라이언트 로컬 추론, 무료, 외부 호출 없음 |
| LLM  | OpenAI GPT-4o-mini              | 회의 중 안건별 요약, 회의 후 종합 정리·다음 회의 안건 생성 | 회의 중 안건당 1회 + 회의 후 2회 |

## 인프라 (AWS)

**Start AWS 학생 활동 프로그램** 제약 하에서 운영. 상세는 [10. AWS 인프라 & 제약](10-AWS-인프라-제약.md) 참조.

| 영역 | 선택 |
| --- | --- |
| 컴퓨트 | EC2 1대 (t3.small), Docker로 NestJS 운영 |
| DB | RDS MySQL 프리티어 (퍼블릭 액세스 허용, EC2 보안 그룹에서만 3306 인입) |
| 캐시 | 별도 매니지드 캐시 없음 — NestJS 인메모리 |
| 도메인/HTTPS | 사용 안 함 — EC2 퍼블릭 IP + HTTP/WS (Electron 데스크탑 앱) |
| 로깅 | EC2 stdout → CloudWatch Logs |
| 배포 | 수동 (`git pull && docker compose up -d --build`) — Access Key 발급 불가 |
| 리전 | ap-northeast-2 (서울). Bedrock 사용 시 us-east-1 |
| Role | EC2 인스턴스 프로필 `SafeInstanceProfileForUser-{username}` |

## 모드 분리

- **회의 중 모드**: 폭 400px 사이드 창, 곁눈질로 한눈에 파악 가능한 디자인
- **회의 외 모드**: 풀 화면 대시보드

## 음성 데이터 처리 원칙

- 음성 파일은 어디에도 저장하지 않음 — 텍스트만 보관
- STT 추론이 클라이언트 로컬에서 일어나므로 **음성이 기기 밖으로 전송되지 않음** (외부 STT 서버 경유 없음)
- 서버로 전송·저장되는 것은 변환 완료된 텍스트뿐
